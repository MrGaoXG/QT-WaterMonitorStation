既然手动运行脚本 ~/run_station.sh 能够成功在屏幕上显示，但重启后却不行，说明问题 100% 出在 Linux 的自启动调用机制 上。树莓派的图形化自启动在最新版本的系统（基于 Wayland / Bookworm）中发生了一些底层变化。

为了彻底解决这个问题，我们将放弃依赖图形界面的 .desktop 文件，改用 最稳定、最底层的系统级服务 (Systemd) 来进行自启动 。

请在 SSH 终端中严格按照以下步骤操作：

### 第一步：清理之前失败的配置
为了防止冲突，先把之前建立的无效快捷方式删掉：

```
rm ~/.config/autostart/watermonitor.
desktop
rm ~/.config/autostart/
water_monitor.desktop
```
### 第二步：创建一个专门用于自启动的包装脚本
Systemd 启动时环境变量极其干净（没有 DISPLAY 等），我们需要写一个能够主动“寻找”桌面的脚本。

执行以下命令直接生成 /home/mrgao/start_gui.sh ：

```
cat << 'EOF' > /home/mrgao/
start_gui.sh
#!/bin/bash
# 等待 X11 / Wayland 服务完全启动
sleep 15

# 强制设置图形环境变量
export DISPLAY=:0
export XAUTHORITY=/home/mrgao/.
Xauthority
export WAYLAND_DISPLAY=wayland-1
export XDG_RUNTIME_DIR=/run/user/
1000

# 切换目录并启动
cd /home/mrgao/WaterMonitorStation/
QT-WaterMonitorStation
./WaterMonitorStation > /home/mrgao/
station_log.txt 2>&1
EOF
```
赋予执行权限：

```
chmod +x /home/mrgao/start_gui.sh
```
### 第三步：创建 Systemd 用户级服务（最可靠的方法）
Systemd 用户服务是在用户登录时触发的，这正是我们需要的。

1. 创建用户服务目录 ：
```
mkdir -p ~/.config/systemd/user/
```
2. 写入服务配置文件 ：
```
cat << 'EOF' > ~/.config/systemd/
user/watermonitor.service
[Unit]
Description=Water Monitor Station 
GUI
After=graphical-session.target
Wants=graphical-session.target

[Service]
Type=simple
ExecStart=/home/mrgao/start_gui.sh
Restart=on-failure
RestartSec=5
Environment=DISPLAY=:0

[Install]
WantedBy=default.target
EOF
```
### 第四步：启用并启动服务
依次执行以下命令，让系统接管这个程序：

```
# 重新加载 Systemd 用户配置
systemctl --user daemon-reload

# 设置开机自启
systemctl --user enable 
watermonitor.service

# 立即尝试启动服务
systemctl --user start watermonitor.
service
```
### 验证方法
执行完 start 命令后：

1. 看屏幕 ：如果此时 HDMI 屏幕亮起了您的程序，说明服务配置完全正确。
2. 查状态 ：在终端输入 systemctl --user status watermonitor.service 。如果是绿色的 active (running) ，说明一切正常。
如果确认运行成功，您可以直接执行 sudo reboot ，重启后只要树莓派进入了桌面，这个服务就会在后台稳定地把您的地面站界面拉起来！如果再失败，通过 cat ~/station_log.txt 就能看到最直接地看到底层阻碍它的报错原因。
